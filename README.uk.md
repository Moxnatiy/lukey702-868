# Lukey 702 / 868 — відкрита прошивка

Реверс-інжиніринг і переписана відкрита прошивка для паяльної станції **Lukey 702**
(на платі маркування **Lukey 868**), SMD rework. У станції **два мікроконтролери**
(обидва ATmega8L): один керує **феном (гарячим повітрям)**, інший — **паяльником**.
Реалізовано обидва.

> 🇬🇧 [Read in English](README.md)

Обидва заводські МК **заблоковані від читання** (`lock=0xFC`), тож оригінальну прошивку зчитати
неможливо — ISP повертає лише «адресну пилку». Тому кожну прошивку відтворено з нуля: розводку
пінів відновлено зі схеми та з дизасемблера референсної прошивки, далі — чиста задокументована
реалізація на C із меню калібрування прямо на станції.

## Структура репозиторію

| Тека | МК | Стан |
|------|-----|------|
| [`hot-air/`](hot-air/) | Фен — ATmega8L | ✅ працює: PID + авто-тюн, меню CAL, відкалібровано, прошито й протестовано |
| [`solder-iron/`](solder-iron/) | Паяльник — ATmega8L | ✅ працює: PID + авто-тюн, меню CAL, температуру відкалібровано, прошито |

Кожен підпроєкт самодостатній (код, документація, Makefile, конфіг VS Code). Драйвер індикатора
(`src/display.c`) спільний **дослівно** — обидві плати мають однакову розводку BDD-4301.

## Спільні можливості

- **PID**-керування нагрівом на 20 Гц з **умовним інтегруванням** (anti-windup) — без перельоту
  при розігріві з холодного.
- **Авто-налаштування PID в один дотик** — релейний метод Åström–Hägglund на 250 °C: вимірює період
  і амплітуду автоколивань, рахує граничне підсилення й застосовує правила Tyreus–Luyben, після
  чого сам зберігає P/I/d.
- **Меню калібрування прямо на станції**, без перекомпіляції: утримати **UP** при вмиканні → `CAL`.
  Короткі UP/DOWN = навігація / ±1, UP-2 с = увійти, DOWN-2 с = назад / вихід.
  Усередині меню тріак нагрівача примусово вимкнено (крім авто-тюну).
- Якісний АЦП: прескалер /128, 16× усереднення, оновлення температури 20 Гц.
- Уставка запам'ятовується в EEPROM; fuse **EESAVE** (`hfuse=0xD1`) зберігає калібрування при
  перепрошивці.
- Кнопки з автоповтором (пауза 0.5 с, далі 20 кроків/с).

## Прошивка фена

- Кінцевий автомат **standby → робота → охолодження** за офіційним 10-пунктним алгоритмом.
- Керування вентилятором/засувкою живлення на PC3, детект підставки на PB4.
- Уставка **100–480 °C**.
- Меню CAL: `P` `I` `d` `OF` (зсув АЦП) `LO` (температура завершення охолодження) `TST` `AT`.
- **Крапки:** усі три світять постійно, поки фен на підставці; під час нагріву остання крапка
  блимає пропорційно потужності ШІМ (швидко = повна потужність).

## Прошивка паяльника

- Єдиний режим роботи — у цього МК **немає вентилятора, засувки живлення й геркона підставки**
  (пін 18 посаджено на землю, `DDRC=0x00`), тож він просто тримає уставку.
- Нагрівач через MOC3023 → тріак **BT135** (~26 В); термопара через каскад **OP07** усередині
  модуля LUKEY 937D → ADC0.
- Уставка **200–480 °C** плюс жорстке аварійне відсічення на **500 °C**.
- Меню CAL: `P` `I` `d` `OF` `Ad` `TST` `AT`, де **`Ad`** показує **живий сирий АЦП** — цей пункт
  додано, бо для паяльника немає документованої таблиці напруга↔температура.
- Температуру відкалібровано на залізі за двома точками (raw 0 = 30 °C, raw 535 = 300 °C).
- Остання крапка блимає пропорційно потужності ШІМ.

Готова прошивка прикріплена до кожного [релізу](https://github.com/Moxnatiy/lukey702-868/releases).

## Залізо

- **МК:** ATmega8L-8PU, внутрішній RC 8 МГц (без кварцу), AREF ≈ 2.5 В (TL431).
- **Плата:** Scorpio «LUKEY868(702)» — аркуші SMD REWORK і SOLDERING.
- **Індикатор:** BDD-4301, 3 розряди, спільний анод (сегменти на PORTD, розряди DG1=PB0 DG2=PB7 DG3=PB6).
- **Програматор:** USBASP (ISP), прошивка через `avrdude`.
- **Тулчейн:** `avr-gcc` (зібрано тулчейном AVR від PlatformIO, gcc 7.3.0).

## Швидкий старт

```bash
cd hot-air        # або: cd solder-iron
make              # зібрати -> fan.hex / iron.hex
make flash        # chip-erase (заодно знімає заводський lock) + прошивка через USBASP
make fuses        # lfuse=0x24 hfuse=0xD1 (EESAVE)
make flash-keep   # згодом: оновити flash, зберігши калібрування
```

Деталі, карти пінів і меню калібрування — у [`hot-air/README.md`](hot-air/README.md) та
[`solder-iron/README.md`](solder-iron/README.md).
